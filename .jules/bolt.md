## 2025-05-25 - [TestRunner Performance Optimization]
**Learning:** In components with a high-frequency timer (e.g., 1s tick), small $O(N)$ calculations and child component re-renders accumulate, causing frame drops or high CPU usage. Memoization of status arrays and stable function references are critical when the timer state lives in the parent component.
**Action:** Always wrap heavy list processing in `useMemo` and stable event handlers in `useCallback` when a sibling state (like a countdown) updates frequently. Use `React.memo` for large child components like palettes/grids that don't depend on the timer.
